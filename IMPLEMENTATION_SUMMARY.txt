1. CREATED: run_baselines.py (457 lines)
   - Single unified script for multi-model, multi-condition QA experiment
   - Supports Flan-T5 (local) + GPT-4 (API optional)
   - Supports both No-RAG and RAG conditions
   - Generates 4 output CSVs for comprehensive metrics

2. KEY FEATURES IMPLEMENTED
   ✅ GPT-4 API integration with conditional execution
   ✅ Explicit skip logic: "Skipping GPT: missing OPENAI_API_KEY"
   ✅ MiniLM-based retriever (cosine similarity, top_k=3)
   ✅ 3-way classification (yes/no/maybe) label extraction
   ✅ Detailed logging: similarity scores, retrieved contexts
   ✅ Confusion matrices per (model, retrieval_setup)
   ✅ Prediction label distribution per (model, retrieval_setup)
   ✅ Macro-averaged F1/precision/recall metrics
   ✅ Run ID tracking for experiment reproducibility

3. OUTPUT FILES GENERATED
   
   a) results/all_experiments.csv (100 * 2-4 models = 200-400 rows)
      Columns: run_id, timestamp, model, retrieval_setup, sample_id,
               question, true_label, prediction_label, correct,
               generated_answer, token_length, top1_similarity,
               mean_similarity, retrieved_contexts
   
   b) results/metrics_summary.csv (2-4 rows)
      Columns: model, retrieval_setup, num_samples, accuracy, macro_f1,
               precision_macro, recall_macro
   
   c) results/confusion_matrices.csv
      Columns: model, retrieval_setup, true_label, predicted_label, count
   
   d) results/prediction_distributions.csv
      Columns: model, retrieval_setup, prediction_label, count, percentage

4. EXPERIMENTAL CONDITIONS
   
   Flan-T5-base (Always runs):
   ├─ No-RAG: Question → Flan-T5 → Answer (baseline)
   └─ RAG:    Question → Retriever → Context → Flan-T5 → Answer
   
   GPT-4 (Conditional, requires OPENAI_API_KEY):
   ├─ No-RAG: Question → GPT-4 → Answer
   └─ RAG:    Question → Retriever → Context → GPT-4 → Answer

5. DEPENDENCIES

6. API INTEGRATION NOTES

   - Proper error handling for API failures

7. RETRIEVAL LOGIC
   - Model: all-MiniLM-L6-v2 (384-dimensional embeddings)
   - Method: Cosine similarity (sklearn, more stable than FAISS)
   - Index: Full PubMedQA dataset (500 contexts)
   - Top-K: 3 contexts per question
   - Output: Concatenated contexts + top1_similarity + mean_similarity

8. LABEL EXTRACTION
   Heuristic: Look for first occurrence of 'yes', 'no', 'maybe' in answer
   Default: 'maybe' (conservative when ambiguous)
